# 文档 document 抽取结果查看页面-修复

此文档描述了 document 抽取成功后点击查看详情后的页面修复内容。

目前发现了以下问题：

## 独立路由改为 dialog 展示
- 原因：进入文档详情返回后又重新刷新 documents 页面，对于高频操作的话体验不友好；
- 修改思路：点击文档详情后不需要进入新的路由，直接在 Documents 页面上弹全屏 Dialog 展示，需要有关闭按钮。

## 文档预览问题
- 问题 1：页面中间的预览图片将所有的图片都展示出来了，仅需要展示一张
- 问题 2：点击最左侧的文档 Page 缩略图，并没有切换，需要切换到指定 Page

## Visualizer 问题
- 问题 1：当前图片页面并没有绘制框选识别出的所有结果，需要默认用浅主题色框出抽取出的所有结果，点击后框的颜色加深，同事右侧的 Extracted Data 自动定位选中；
- 问题 2：点击右侧 Extracted Data 区域，数据有高亮，但是并没有在中间的 Page 预览图中框出，也就是中间的预览区与右侧的 Extracted Data 需要联动；
- 问题 3：预览区域顶部没有放大、缩小、旋转、恢复按钮，需要有这样的操作按钮

## 给你的提示
- 你注意看 Visualizer 的文档，以及 textin-extract-v3.md 的说明，返回的数据可能需要进行一个转换处理，因为坐标的单位不一样？我也不太确定。你需要研究一下。
