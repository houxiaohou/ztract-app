# 项目初始化需求文档

此文档描述了创建新项目后的初始化流程。

## 流程
1. 用户输入项目名称 Name、描述 Description 后创建项目；
2. 创建成功后路由自动切换至项目详情页；
3. 如果该项目还没有 schema，不管是不是新创建，只要没有 schema 就显示初始化页面；
4. 初始化页面引导用户添加 schema，保存 schema 成功后再回到 project 的 documents 页面；

## 参考接口

请参考 `@docs/api.json` 文件中相关 schema 的接口。

## 添加 schema 的方式

有三种添加 schema 的方式，基本流程都是根据第一步的结果（例如：选择模板、用户输入自然语言、上传的文件）调用接口返回 schema 草稿，用户可以再编辑，然后保存。
关于 schema 的编辑功能，已经在项目 schema 管理那个页面完成了，你可以考虑一下如何进行复用的问题，不强求。

### 1. 选择模板
从后台获取的模板选择创建，选择模板后，展示 schema，同时可以修改，确认后保存。

### 2. 输入 prompt
通过一个输入框，让用户输入需要从文档抽取什么内容，调用接口，完成解析，展示 schema，同时可以修改，确认后保存。

### 3. 上传文件
通过一个上传文件功能，上传文件获取 document_id，调用接口，完成解析，展示 schema，同时可以修改，确认后保存。
这个 document_id 也需要关联到此项目，后续也会自动解析该文件的内容。

## 项目初始化页面布局
1. 可以有一个单独的路径，比如 /projects/:id/init
2. 左侧导航栏可以隐藏，但是有点奇怪
3. 具体你发挥吧，我没太想好具体这个页面的路由、页面布局。
